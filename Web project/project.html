Why Sparx password policies
aren't enough
  The password policies cannot be applied to specific OUs.   Dictionary words, patterns, and palindromes cannot be restricted.   Consecutive repetition of the same character cannot be prevented.   The password policy cannot be enforced during password reset by admins in the user and computers(ADUC).   The password must meet complexity requirments policy setting cannot mandate the number of characters from a certain character type.
